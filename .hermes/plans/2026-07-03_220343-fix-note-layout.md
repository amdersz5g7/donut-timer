# Fix TimerForm Layout — Notes Alignment

> **For Hermes:** Execute directly — single task, single file.

**Goal:** Fix misaligned layout where the notes textarea breaks the Bootstrap grid row, causing uneven spacing between form controls.

**Architecture:** Split the single `<div class="row">` in TimerForm into two rows: row 1 for inputs + button (original layout), row 2 for notes textarea (new). This keeps Bootstrap column math correct and aligns all elements properly.

**Tech Stack:** Bootstrap 3 grid (via mini.css), Svelte 3

---

## Current Problem

`src/components/TimerForm.svelte` has all four elements in ONE `<div class="row">`:

```svelte
<div class="row">
  <div class="col-sm-3"> Minutes </div>   <!-- 3/12 -->
  <div class="col-sm-3"> Items </div>     <!-- 3/12 = 6/12 total -->
  <div class="col-sm-12"> Notes </div>    <!-- 12/12 → WRAPS to next line -->
  <div class="col-sm-6"> Add Button </div><!-- 6/12 → wraps again, alone -->
</div>
```

Visual result:
```
[Minutes    | Items    ]
[Notes (full width)      ]
[         Add Timer       ]
```

This causes:
- Notes unaligned with Minutes/Items (different width, different row)
- Add Timer button isolated on its own line without context
- Extra vertical spacing that looks broken

## Fix

Split into two rows — inputs + button in row 1, notes in row 2:

```svelte
<div class="row">
  <div class="col-sm-3"> Minutes </div>
  <div class="col-sm-3"> Items </div>
  <div class="col-sm-6"> Add Button </div>
</div>
<div class="row">
  <div class="col-sm-12"> Notes </div>
</div>
```

Visual result:
```
[Minutes    | Items    |  Add Timer    ]
[Notes (full width)                     ]
```

The button moves back inline with minutes/items, and notes has a clean dedicated row below.
